# 贴贴日记多端适配说明

贴贴日记现在按“一套产品能力，多个端壳”来组织，避免每个端从零重写。

## 端列表

| 端 | 目录 | 定位 | 当前状态 |
| --- | --- | --- | --- |
| 网页端 | `apps/web` | 浏览器直接访问 | Vue 3 主应用 |
| PWA 端 | `apps/web` | iPhone/Android 添加到主屏幕 | 已有 manifest 和 service worker |
| Android 软件端 | `apps/android` | 原生 Android WebView 壳 | 可用 Android Studio 打开，加载 Web/PWA 主应用 |
| 微信小程序端 | `apps/mp-wechat` | 微信内轻量记录 | 原生小程序壳，支持本地缓存、邮箱登录、手动推送/拉取同步、基础 AI 编辑、轻量贴纸画布和卡片导出 |

## 常用命令

```powershell
npm.cmd run dev
npm.cmd run build:web
npm.cmd run build:pwa
npm.cmd run build:android
npm.cmd run build:mp-wechat
npm.cmd run build:clients
```

`build:android` 和 `build:mp-wechat` 当前做结构校验；真正打包分别交给 Android Studio 和微信开发者工具。

## 网页端

网页端就是当前主应用：

```text
http://localhost:5173/
```

生产部署时用 `apps/web/dist` 静态文件，后端接口继续由 `apps/api` 提供。

## PWA 端

PWA 和网页端共用 `apps/web`。PWA 相关文件：

```text
apps/web/public/manifest.webmanifest
apps/web/public/sw.js
apps/web/public/icons/
```

iPhone 上用 Safari 打开正式 HTTPS 地址后，选择“添加到主屏幕”即可。

## Android 软件端

Android 端目录：

```text
apps/android
```

开发时默认加载：

```text
http://10.0.2.2:5173
```

这是 Android 模拟器访问宿主机 Vite 服务的地址。发布前修改：

```text
apps/android/app/build.gradle
```

里的 `TIETIE_WEB_URL` 为正式 HTTPS 域名，然后用 Android Studio 构建 APK/AAB。

## 微信小程序端

小程序端目录：

```text
apps/mp-wechat
```

用微信开发者工具打开该目录。当前小程序端可以先离线写本地日记，也可以在设置页填写后端 API 地址，用邮箱验证码登录后手动同步。

本地开发常用后端地址：

```text
http://127.0.0.1:4000/api
```

真机调试时需要改成电脑局域网 IP 或正式 HTTPS 域名，例如：

```text
http://192.168.x.x:4000/api
https://api.example.com/api
```

已接入能力：

1. 后端邮箱验证码登录。
2. `/api/sync` 推送和拉取日记。
3. `/api/files/upload` 同步前上传本地照片。
4. `/api/ai/generate-diary` 根据照片数量、心情、标签、文风和长度生成正文。
5. `/api/ai/segment` 与 `/api/ai/stylize` 对选中的照片生成贴纸，支持直接点选主体、拖拽框选主体、中心、上方、整张、左侧、右侧主体预设，以及五种贴纸风格。
6. 小程序本地保存 `stickers` 数组，生成后的贴纸可在轻量画布中直接拖动，也可用按钮移动、缩放、旋转、置顶和删除；同步时会保留贴纸变换信息。
7. 小程序编辑页可用本地 canvas 合成卡片图片，包含日期、心情、标签、正文和贴纸排版，生成后可保存到相册。

后续可以继续补卡片导出的更多背景和版式。

## 维护原则

- 复杂编辑器和 AI 能力优先在 `apps/web` 完整实现。
- Android 端优先复用 Web/PWA，不重复实现编辑器。
- 小程序端保留轻量记录体验，云同步和基础 AI 能力走同一套后端，复杂编辑仍优先在 Web/PWA 打磨。
- 共享数据结构继续放在 `packages/shared`，后续小程序可按该类型补一份轻量适配。
