# 部署说明

这份配置用于把贴贴日记部署到自己的服务器：Caddy 负责 HTTPS 和反向代理，前端由 Nginx 托管，后端连接 PostgreSQL，上传图片保存到 Docker volume。

## 准备服务器

- 一台 Linux 服务器，安装 Docker 和 Docker Compose。
- 一个域名，并把域名 A 记录指向服务器 IP。
- 开放 `80` 和 `443` 端口。iPhone 安装 PWA 时，正式环境建议使用 HTTPS 域名。

## 配置环境变量

复制环境变量样例：

```bash
cp deploy/.env.example deploy/.env
```

至少修改这些值：

- `SITE_DOMAIN`：你的域名，例如 `diary.example.com`。
- `PUBLIC_BASE_URL`：完整访问地址，例如 `https://diary.example.com`。
- `POSTGRES_PASSWORD`：数据库密码。
- `DATABASE_URL`：把里面的密码同步改成 `POSTGRES_PASSWORD`。
- `JWT_SECRET`：登录 token 密钥，建议使用长随机字符串。

AI 相关变量先留空也可以，应用会继续使用占位处理和前端兜底体验。

## 启动

```bash
npm run deploy:up
```

查看状态：

```bash
docker compose -p tietie --env-file deploy/.env -f docker-compose.prod.yml ps
npm run deploy:logs
```

健康检查：

```bash
curl https://你的域名/api/health
```

## 数据与备份

生产数据主要在两个 volume：

- `postgres-data`：日记同步数据。
- `api-storage`：上传图片文件。

备份 PostgreSQL：

```bash
docker compose -p tietie --env-file deploy/.env -f docker-compose.prod.yml exec postgres pg_dump -U "$POSTGRES_USER" "$POSTGRES_DB" > tietie-backup.sql
```

图片文件建议定期备份 `api-storage` volume。后续如果迁移到对象存储，只需要替换后端文件存储层和相关环境变量。

## 更新版本

拉取或复制新代码后重新构建：

```bash
npm run deploy:up
```

如果只想看后端日志：

```bash
npm run deploy:logs
```
